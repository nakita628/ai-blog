Faker.js入門完全ガイド：テストデータ生成の定番ライブラリを初心者向けに解説 ​

Faker.jsは、テストや開発時に大量のリアルなダミーデータを生成するためのJavaScriptライブラリです。名前、住所、電話番号、メールアドレスなど、様々な種類のデータを簡単に生成でき、開発効率を大幅に向上させます。初心者にも分かりやすいFaker.jsの基本から実践的な使い方まで詳しく解説します。

Faker.jsとは何か ​

Faker.jsは、開発やテスト時に必要なダミーデータを生成するためのライブラリです。以下の特徴があります：

  • リアルなデータ生成: 実際に存在しそうな名前、住所、電話番号を生成
  • 多様なデータタイプ: 人物、場所、日付、金融、商品、ハッカー用語など
  • 多言語対応: 70以上のロケールに対応
  • ブラウザ・Node.js対応: フロントエンドとバックエンドの両方で使用可能
  • TypeScript対応: 型安全性を提供
  • カスタマイズ可能: 独自のデータ生成ロジックを追加可能

インストール ​

npmを使用したインストール ​

bash
# 開発依存関係としてインストール
npm install --save-dev @faker-js/faker

# または本番依存関係としてインストール
npm install @faker-js/faker

yarnを使用したインストール ​

bash
yarn add -D @faker-js/faker

pnpmを使用したインストール ​

bash
pnpm add -D @faker-js/faker

注文データの生成 ​

typescript
import { faker } from '@faker-js/faker';

function generateOrder() {
  const orderItems = faker.helpers.multiple(() => ({
    productId: faker.string.uuid(),
    productName: faker.commerce.productName(),
    quantity: faker.number.int({ min: 1, max: 10 }),
    price: faker.commerce.price(),
    total: faker.commerce.price()
  }), { count: faker.number.int({ min: 1, max: 5 }) });

  const subtotal = orderItems.reduce((sum, item) => sum + parseFloat(item.total), 0);
  const tax = subtotal * 0.1;
  const shipping = faker.number.float({ min: 0, max: 50, fractionDigits: 2 });
  const total = subtotal + tax + shipping;

  return {
    id: faker.string.uuid(),
    orderNumber: faker.string.alphanumeric(10).toUpperCase(),
    customer: {
      id: faker.string.uuid(),
      name: faker.person.fullName(),
      email: faker.internet.email(),
      phone: faker.phone.number()
    },
    items: orderItems,
    shippingAddress: {
      street: faker.location.streetAddress(),
      city: faker.location.city(),
      state: faker.location.state(),
      zipCode: faker.location.zipCode(),
      country: faker.location.country()
    },
    billingAddress: {
      street: faker.location.streetAddress(),
      city: faker.location.city(),
      state: faker.location.state(),
      zipCode: faker.location.zipCode(),
      country: faker.location.country()
    },
    payment: {
      method: faker.helpers.arrayElement(['credit_card', 'paypal', 'bank_transfer']),
      cardNumber: faker.finance.creditCardNumber(),
      expiryDate: faker.date.future().toISOString().slice(0, 7)
    },
    pricing: {
      subtotal: subtotal.toFixed(2),
      tax: tax.toFixed(2),
      shipping: shipping.toFixed(2),
      total: total.toFixed(2)
    },
    status: faker.helpers.arrayElement(['pending', 'processing', 'shipped', 'delivered', 'cancelled']),
    createdAt: faker.date.past(),
    updatedAt: faker.date.recent()
  };
}

const order = generateOrder();
console.log(order);

import { faker } from '@faker-js/faker';

function generateDateData() {
  return {
    // 過去の日付
    pastDate: faker.date.past(),
    pastYear: faker.date.past({ years: 10 }),
    pastMonth: faker.date.past({ months: 6 }),
    
    // 未来の日付
    futureDate: faker.date.future(),
    futureYear: faker.date.future({ years: 5 }),
    futureMonth: faker.date.future({ months: 3 }),
    
    // 最近の日付
    recentDate: faker.date.recent(),
    recentDays: faker.date.recent({ days: 7 }),
    
    // 特定の範囲
    betweenDates: faker.date.between({
      from: new Date('2020-01-01'),
      to: new Date('2023-12-31')
    }),
    
    // 生年月日
    birthdate: faker.date.birthdate(),
    adultBirthdate: faker.date.birthdate({ min: 18, max: 65, mode: 'age' }),
    
    // 曜日
    weekday: faker.date.weekday(),
    
    // 月
    month: faker.date.month(),
    
    // 年
    year: faker.date.anytime().getFullYear()
  };
}

import { faker } from '@faker-js/faker';

// カスタムデータ生成関数
function generateCustomData() {
  return {
    // ランダムなID生成
    customId: faker.string.alphanumeric(8).toUpperCase(),
    
    // 特定の形式のデータ
    phoneNumber: faker.phone.number('+81-90-####-####'),
    postalCode: faker.location.zipCode('###-####'),

    priority: faker.helpers.arrayElement(['low', 'medium', 'high', 'urgent']),
    
    // 重み付き選択
    category: faker.helpers.weightedArrayElement([
      { weight: 50, value: 'general' },
      { weight: 30, value: 'premium' },
      { weight: 20, value: 'vip' }
    ]),
    
    // 複数の値を生成
    tags: faker.helpers.multiple(() => faker.word.adjective(), { count: 3 }),
    skills: faker.helpers.multiple(() => faker.person.jobType(), { count: 5 }),
    
    // オブジェクトの配列
    addresses: faker.helpers.multiple(() => ({
      type: faker.helpers.arrayElement(['home', 'work', 'billing']),
      street: faker.location.streetAddress(),
      city: faker.location.city(),
      isPrimary: faker.datatype.boolean()
    }), { count: 2 })
  };
}

まとめ ​

Faker.jsは、開発やテスト時に必要なダミーデータを効率的に生成する強力なライブラリです。主なポイントは以下の通りです：

  • 多様なデータタイプ: 人物、場所、日付、金融、商品など様々なデータを生成
  • リアルなデータ: 実際に存在しそうな自然なデータを生成
  • 多言語対応: 70以上のロケールで各国のデータを生成
  • カスタマイズ可能: 独自のデータ生成ロジックを追加可能
  • 再現可能: シード値を使用して同じデータを再生成
  • TypeScript対応: 型安全性を提供

Faker.jsを活用することで、テストデータの準備時間を大幅に短縮し、より現実的なテストシナリオを構築できます。

次のステップ ​

  • より複雑なデータ構造の生成
  • カスタムプロバイダーの作成
  • パフォーマンス最適化
  • データベースシードでの活用
  • テストフレームワークとの連携
  • 本番環境での注意点
