# ブラウザで使える便利なAPI完全ガイド：モダンWeb開発に欠かせない機能

モダンなWebアプリケーション開発では、ブラウザが提供するネイティブAPIを活用することで、より豊かなユーザー体験を実現できます。この記事では、実務で頻繁に使われる便利なブラウザAPIを、実際のコード例と共に詳しく解説します。

## 1. Intersection Observer API

要素がビューポートに入ったかどうかを効率的に監視できるAPIです。スクロールアニメーションや遅延読み込みに最適です。

### 基本的な使い方

```typescript
const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        // 要素がビューポートに入った時の処理
        entry.target.classList.add('visible');
      }
    });
  },
  {
    root: null, // ビューポートを基準にする
    rootMargin: '0px',
    threshold: 0.1, // 10%見えたら発火
  }
);

// 監視対象の要素を登録
const target = document.querySelector('.lazy-element');
if (target) {
  observer.observe(target);
}
```

### 画像の遅延読み込み実装

```typescript
class LazyImageLoader {
  private observer: IntersectionObserver;

  constructor() {
    this.observer = new IntersectionObserver(
      (entries) => {
        entries.forEach((entry) => {
          if (entry.isIntersecting) {
            const img = entry.target as HTMLImageElement;
            const src = img.dataset.src;
            if (src) {
              img.src = src;
              img.removeAttribute('data-src');
              this.observer.unobserve(img);
            }
          }
        });
      },
      { rootMargin: '50px' }
    );
  }

  observe(img: HTMLImageElement): void {
    this.observer.observe(img);
  }

  disconnect(): void {
    this.observer.disconnect();
  }
}

// 使用例
const loader = new LazyImageLoader();
document.querySelectorAll('img[data-src]').forEach((img) => {
  loader.observe(img as HTMLImageElement);
});
```

## 2. Fetch API

XMLHttpRequestのモダンな代替として、PromiseベースでHTTPリクエストを送信できます。

### 基本的な使い方

```typescript
async function fetchData(url: string): Promise<unknown> {
  try {
    const response = await fetch(url);
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    return await response.json();
  } catch (error) {
    console.error('Fetch error:', error);
    throw error;
  }
}
```

### 型安全なFetchラッパー

```typescript
type FetchOptions = RequestInit & {
  timeout?: number;
};

async function fetchWithTimeout(
  url: string,
  options: FetchOptions = {}
): Promise<Response> {
  const { timeout = 5000, ...fetchOptions } = options;

  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), timeout);

  try {
    const response = await fetch(url, {
      ...fetchOptions,
      signal: controller.signal,
    });
    return response;
  } finally {
    clearTimeout(timeoutId);
  }
}

// 使用例
const data = await fetchWithTimeout('/api/users', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ name: 'John' }),
  timeout: 3000,
});
```

## 3. Clipboard API

クリップボードへのコピー・貼り付けを簡単に実装できるAPIです。

### テキストのコピー

```typescript
async function copyToClipboard(text: string): Promise<boolean> {
  try {
    await navigator.clipboard.writeText(text);
    return true;
  } catch (error) {
    console.error('Failed to copy:', error);
    return false;
  }
}

// 使用例
const button = document.querySelector('.copy-button');
button?.addEventListener('click', async () => {
  const success = await copyToClipboard('コピーするテキスト');
  if (success) {
    console.log('コピー成功');
  }
});
```

### 画像のコピー

```typescript
async function copyImageToClipboard(imageUrl: string): Promise<boolean> {
  try {
    const response = await fetch(imageUrl);
    const blob = await response.blob();
    await navigator.clipboard.write([
      new ClipboardItem({ [blob.type]: blob }),
    ]);
    return true;
  } catch (error) {
    console.error('Failed to copy image:', error);
    return false;
  }
}
```

## 4. Web Storage API

`localStorage`と`sessionStorage`を使用して、ブラウザにデータを保存できます。

### 型安全なStorageラッパー

```typescript
class TypedStorage<T> {
  constructor(private key: string, private storage: Storage = localStorage) {}

  get(): T | null {
    const item = this.storage.getItem(this.key);
    if (!item) return null;

    try {
      return JSON.parse(item) as T;
    } catch {
      return null;
    }
  }

  set(value: T): void {
    this.storage.setItem(this.key, JSON.stringify(value));
  }

  remove(): void {
    this.storage.removeItem(this.key);
  }

  clear(): void {
    this.storage.clear();
  }
}

// 使用例
type UserPreferences = {
  theme: 'light' | 'dark';
  language: 'ja' | 'en';
};

const userPrefs = new TypedStorage<UserPreferences>('user-preferences');
userPrefs.set({ theme: 'dark', language: 'ja' });
const prefs = userPrefs.get();
```

## 5. Geolocation API

ユーザーの位置情報を取得できるAPIです。

### 位置情報の取得

```typescript
type Position = {
  latitude: number;
  longitude: number;
  accuracy: number;
};

function getCurrentPosition(): Promise<Position> {
  return new Promise((resolve, reject) => {
    if (!navigator.geolocation) {
      reject(new Error('Geolocation is not supported'));
      return;
    }

    navigator.geolocation.getCurrentPosition(
      (position) => {
        resolve({
          latitude: position.coords.latitude,
          longitude: position.coords.longitude,
          accuracy: position.coords.accuracy,
        });
      },
      (error) => {
        reject(error);
      },
      {
        enableHighAccuracy: true,
        timeout: 5000,
        maximumAge: 0,
      }
    );
  });
}

// 使用例
try {
  const position = await getCurrentPosition();
  console.log(`緯度: ${position.latitude}, 経度: ${position.longitude}`);
} catch (error) {
  console.error('位置情報の取得に失敗:', error);
}
```

## 6. Web Workers

メインスレッドをブロックせずに、バックグラウンドで処理を実行できます。

### Workerの作成と使用

```typescript
// worker.ts
self.onmessage = (e: MessageEvent<{ data: number[] }>) => {
  const { data } = e.data;
  const result = data.map((n) => n * 2);
  self.postMessage({ result });
};

// main.ts
const worker = new Worker(new URL('./worker.ts', import.meta.url), {
  type: 'module',
});

worker.postMessage({ data: [1, 2, 3, 4, 5] });

worker.onmessage = (e: MessageEvent<{ result: number[] }>) => {
  console.log('結果:', e.data.result);
  worker.terminate();
};
```

## 7. File API

ファイルの読み取りや操作を行うAPIです。

### ファイルの読み取り

```typescript
function readFileAsText(file: File): Promise<string> {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = (e) => {
      resolve(e.target?.result as string);
    };
    reader.onerror = reject;
    reader.readAsText(file);
  });
}

function readFileAsDataURL(file: File): Promise<string> {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = (e) => {
      resolve(e.target?.result as string);
    };
    reader.onerror = reject;
    reader.readAsDataURL(file);
  });
}

// 使用例
const input = document.querySelector<HTMLInputElement>('input[type="file"]');
input?.addEventListener('change', async (e) => {
  const file = e.target.files?.[0];
  if (file) {
    const text = await readFileAsText(file);
    console.log('ファイル内容:', text);
  }
});
```

## 8. Notification API

ブラウザ通知を表示できるAPIです。

### 通知の表示

```typescript
async function requestNotificationPermission(): Promise<NotificationPermission> {
  if (!('Notification' in window)) {
    throw new Error('このブラウザは通知をサポートしていません');
  }

  if (Notification.permission === 'default') {
    await Notification.requestPermission();
  }

  return Notification.permission;
}

async function showNotification(title: string, options?: NotificationOptions): Promise<void> {
  const permission = await requestNotificationPermission();

  if (permission === 'granted') {
    new Notification(title, {
      body: options?.body,
      icon: options?.icon,
      badge: options?.badge,
      tag: options?.tag,
      requireInteraction: options?.requireInteraction,
    });
  }
}

// 使用例
await showNotification('新しいメッセージ', {
  body: 'メッセージが届きました',
  icon: '/icon.png',
  tag: 'message',
});
```

## 9. ResizeObserver API

要素のサイズ変更を監視できるAPIです。

### 要素サイズの監視

```typescript
function observeResize(
  element: Element,
  callback: (entries: ResizeObserverEntry[]) => void
): ResizeObserver {
  const observer = new ResizeObserver(callback);
  observer.observe(element);
  return observer;
}

// 使用例
const box = document.querySelector('.resizable-box');
if (box) {
  observeResize(box, (entries) => {
    const entry = entries[0];
    const { width, height } = entry.contentRect;
    console.log(`サイズ変更: ${width}x${height}`);
  });
}
```

## 10. MutationObserver API

DOMの変更を監視できるAPIです。

### DOM変更の監視

```typescript
function observeMutations(
  target: Node,
  callback: (mutations: MutationRecord[]) => void
): MutationObserver {
  const observer = new MutationObserver(callback);
  observer.observe(target, {
    childList: true,
    subtree: true,
    attributes: true,
    attributeOldValue: true,
  });
  return observer;
}

// 使用例
const container = document.querySelector('.container');
if (container) {
  observeMutations(container, (mutations) => {
    mutations.forEach((mutation) => {
      if (mutation.type === 'childList') {
        console.log('子要素が変更されました');
      }
    });
  });
}
```

## 実践的な組み合わせ例

### 無限スクロールの実装

```typescript
class InfiniteScroll {
  private observer: IntersectionObserver;
  private loading = false;

  constructor(
    private container: HTMLElement,
    private loadMore: () => Promise<void>
  ) {
    this.observer = new IntersectionObserver(
      async (entries) => {
        if (entries[0].isIntersecting && !this.loading) {
          this.loading = true;
          await this.loadMore();
          this.loading = false;
        }
      },
      { rootMargin: '100px' }
    );
  }

  start(): void {
    const sentinel = document.createElement('div');
    sentinel.className = 'scroll-sentinel';
    this.container.appendChild(sentinel);
    this.observer.observe(sentinel);
  }
}
```

## まとめ

ブラウザネイティブAPIを活用することで、外部ライブラリに依存せずに、パフォーマンスの高い機能を実装できます。

**主要なポイント**:
- **Intersection Observer**: スクロールアニメーションや遅延読み込みに最適
- **Fetch API**: モダンなHTTP通信の標準
- **Clipboard API**: コピー・貼り付け機能の実装が簡単
- **Web Storage**: クライアント側のデータ永続化
- **Geolocation API**: 位置情報ベースの機能実装
- **Web Workers**: 重い処理をバックグラウンドで実行
- **File API**: ファイル操作の基本
- **Notification API**: ブラウザ通知の表示
- **ResizeObserver / MutationObserver**: DOM変更の効率的な監視

これらのAPIを適切に組み合わせることで、ユーザー体験の向上とパフォーマンスの最適化を両立できます。
